Feature

Facilitating instant reporting of Emergency cases

v. 2.8 For Mac and Android

This application is developed for Emergency Medical Technicians (EMTs) to report new emergency cases while they are travelling to the nearest hospital. The application will provide real time updates to the clinical web application that will be accessed by the doctors at the hospital. The real time updates will include the type emergency (Trauma\Stroke or MI), the status of the patient with details of the medical status and the location updates. This will help the doctors at the hospital to be aware of the situation and be prepared to receive the patient upon arrival.

Report Emergency

Add the emergency report by the mobile user (EMT) to the DB, which in return can be viewed by the doctors using the Clinical Web App

    Capture Basic Info Like:
  • The current location of the EMT
  • Emergency Type: Trauma\ Stroke or MI
  • Patient Basic Infor - Name, Age, Sex
  • Description of the incident
  • Location of the incident
  • Details of the Patient status - The exact details to be captured and stored need to be discusses and finalized.

VIEWING TRAUMA CASE DETAILS

It will capture the set of emergency details (patient status details).

  • The details to be captured will differ based on the emergency type selected (Trauma\Stroke or MI). The details for each option will need to be discussed and clarified with HCI. For example: for a trauma this will capture the last vitals, Mechanism of injury etc.
  • Furthermore, the screen will provide a model image a male\female (depending on the patient's sex) for the EMT to provide further information.
  • Upon clicking on the image and selecting the options provided the user will be directed to the second screen described next. Note: this needs to be clarified further.

Track EMT Location

Another feature of this app is to track the EMT Location so that the doctors will have an understanding of the arrival time

  • This app will update the EMT's location from the moment the new emergency is reported until it is marked as closed. The mobile app will make calls to the API every 1 mile so the doctors will have an understanding of the arrival time
  • System automatically identify the location of the case and recorded it within the case details.

Other Features

  • Instant communication between the EMT users and the Clinical backend doctors.
  • Complete clinical diagnostic for the case by the EMT users before going to the hospital.
  • The solution utilizes available technology efficiently to reduce the risk in saving the emergency cases.
  • Allow the Emergency treatment operation to operate faster.
  • Effective use of location identification available in mobile devices.
  • Every Expert in the system EMT user and Clinical user will have his/her own profile that represents his / her role in the system with a facility to update this profile.
  • The solution helps clinical users to determine the different correct preparation steps required to receive the cases at the hospital and treat them which will highly help in saving the cases life.
  • Reported cases can be filtered based on type , date & time , EMT users , area, and current status.
  • System admin has full control on the reported cases in the system and the admin has the facility to report new cases (Trauma or high Acuity) from the admin web panel

MEDICAL TRAUMA ANDROID APPLICATION

  • Allowing Emergency Medical Technicians to report new emergency cases during transport
  • Providing real time updates to clinical web application which is accessed by doctors at hospital
